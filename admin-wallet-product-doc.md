# Admin 钱包后台产品文档

## 1. 产品概述

### 1.1 产品名称

Admin 钱包后台 / Wallet Admin Console

### 1.2 产品定位

Admin 钱包后台用于平台内部团队管理 creator / publisher / 合作方的钱包资产、收益结算、提现审核、出款状态、风控冻结、人工调账、财务对账和操作审计。

Creator 端关注“我赚了多少钱、可以提现吗、提现到哪里”；Admin 端关注“平台如何确认这笔钱、是否允许提现、是否成功出款、异常资金如何处理、后续如何对账和追责”。

核心链路：

```text
收益产生 -> 平台审核 -> 结算入账 -> 用户提现 -> Admin 审核 -> 第三方出款 -> 对账确认 -> 审计留痕
```

### 1.3 目标用户

- 运营人员
- 财务人员
- 风控人员
- 客服人员
- 业务管理员
- 超级管理员
- 审计/合规人员

### 1.4 核心价值

- 让平台可视化掌握全局资金状态。
- 标准化收益结算、提现审核和出款流程。
- 支持异常资金冻结、扣减、补偿和冲正。
- 降低重复出款、错误打款、异常提现和账务不平风险。
- 为财务对账、合规审计和用户申诉提供完整证据链。

## 2. 业务目标

### 2.1 业务目标

- 提升提现审核和出款处理效率。
- 降低提现失败率和人工沟通成本。
- 提高账务准确性，确保平台账、服务商账、银行账一致。
- 建立完整的资金操作权限体系和审计日志。
- 支持后续多币种、多国家、多付款服务商扩展。

### 2.2 管理目标

Admin 需要能回答以下问题：

- 平台当前有多少待结算金额？
- 有多少金额已经到账但尚未提现？
- 有多少提现申请等待审核？
- 哪些提现存在风险？
- 哪些出款失败了，失败原因是什么？
- 哪些用户钱包被冻结？
- 哪些账单和第三方服务商对不上？
- 哪些资金被人工调整过，谁操作的，为什么？

## 3. 角色与权限

### 3.1 角色定义

| 角色 | 说明 |
| --- | --- |
| 运营 | 查看钱包、处理普通用户咨询、发起结算或调账申请 |
| 客服 | 查询用户钱包、提现状态、失败原因，不允许改动资金 |
| 财务 | 审核结算单、审核提现、查看财务流水、执行对账 |
| 风控 | 查看风险命中、冻结钱包、解冻钱包、拒绝高风险提现 |
| 管理员 | 管理配置、查看全量数据、处理高级操作 |
| 超级管理员 | 执行高风险动作，如大额调账、批量出款、权限分配 |
| 审计员 | 只读查看资金流水、操作日志、对账结果 |

### 3.2 权限矩阵

| 功能 | 运营 | 客服 | 财务 | 风控 | 管理员 | 超级管理员 | 审计员 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 查看钱包概览 | 是 | 是 | 是 | 是 | 是 | 是 | 是 |
| 查看用户钱包 | 是 | 是 | 是 | 是 | 是 | 是 | 是 |
| 查看敏感账户信息 | 否 | 否 | 部分 | 部分 | 部分 | 是 | 否 |
| 审核提现 | 否 | 否 | 是 | 可拒绝 | 是 | 是 | 否 |
| 执行出款 | 否 | 否 | 是 | 否 | 是 | 是 | 否 |
| 冻结/解冻钱包 | 否 | 否 | 否 | 是 | 是 | 是 | 否 |
| 人工调账 | 发起 | 否 | 审批 | 发起扣减 | 审批 | 执行 | 否 |
| 导出数据 | 否 | 否 | 是 | 是 | 是 | 是 | 是 |
| 查看操作日志 | 否 | 否 | 是 | 是 | 是 | 是 | 是 |

### 3.3 权限原则

- 资金变更类操作必须有明确权限。
- 高风险操作必须二次确认。
- 大额调账、批量出款、钱包冻结建议采用审批流。
- 敏感收款信息默认脱敏展示。
- 所有关键操作必须记录操作日志。

## 5. Admin 核心业务链路

### 5.1 收益结算链路

```text
业务系统生成收益
-> 收益进入待结算
-> Admin 查看待结算明细
-> 系统生成结算单
-> 财务/运营审核结算单
-> 确认入账
-> 用户钱包可提现余额增加
-> 生成入账流水
```

### 5.2 提现审核链路

```text
用户提交提现申请
-> 系统冻结提现金额
-> 提现单进入待审核
-> Admin 查看提现详情和风险信息
-> 审核通过或拒绝
-> 通过后进入待出款
-> 拒绝后释放冻结金额
```

### 5.3 出款链路

```text
提现审核通过
-> 创建出款任务
-> 调用第三方付款服务商
-> 服务商受理
-> 出款处理中
-> 出款成功 / 出款失败
-> 更新提现单状态
-> 生成出款流水
```

### 5.4 对账链路

```text
导入服务商账单
-> 获取平台提现流水
-> 自动匹配交易号、金额、币种、状态
-> 生成对账结果
-> 发现差异
-> 创建差错单
-> 人工处理
-> 关闭差错
```

## 6. 首页资金概览

### 6.1 页面目标

让 Admin 快速掌握平台整体资金状态和待处理事项。

### 6.2 核心指标

| 指标 | 定义 |
| --- | --- |
| 累计收益总额 | 平台历史确认的 creator 收益总额 |
| 待结算金额 | 已产生但尚未入账的钱 |
| 已到账金额 | 已完成结算并进入用户钱包的钱 |
| 可提现金额 | 当前所有用户可申请提现的总额 |
| 提现中金额 | 已冻结但尚未最终出款完成的钱 |
| 已提现金额 | 历史成功出款金额 |
| 提现失败金额 | 出款失败且已退回或待处理金额 |
| 冻结金额 | 因提现、风控或争议被冻结的钱 |
| 扣减金额 | 违规、退款、修正产生的扣减总额 |
| 待审核提现数 | 当前等待 Admin 审核的提现申请数量 |
| 异常账户数 | 命中风控或被冻结的钱包账户数量 |

### 6.3 筛选维度

- 时间范围
- 国家/地区
- 币种
- 业务线
- 收益来源
- 用户类型
- 提现渠道
- 风险等级

### 6.4 待办入口

首页应提供待办卡片：

- 待审核提现
- 出款失败
- 待审核结算单
- 对账差异
- 高风险提现
- 待处理调账申请

## 7. Creator 钱包查询

### 7.1 页面目标

支持 Admin 查询任意 creator 的钱包资产、收益流水、提现记录和账户状态。

### 7.2 查询条件

- 用户 ID
- 用户邮箱
- 昵称
- PayPal 邮箱
- 银行账户尾号
- 钱包状态
- 国家/地区
- 注册时间

### 7.3 用户钱包详情

钱包详情应展示：

| 字段 | 说明 |
| --- | --- |
| 用户 ID | 平台用户唯一标识 |
| 钱包 ID | 钱包账户唯一标识 |
| 钱包状态 | 正常 / 冻结 / 限制提现 / 注销 |
| 币种 | 默认 USD |
| 累计收益 | 历史确认收益 |
| 待结算金额 | 尚未入账收益 |
| 可提现余额 | 当前可发起提现金额 |
| 冻结金额 | 提现中或风控冻结金额 |
| 已提现金额 | 历史成功提现金额 |
| 扣减金额 | 历史扣减金额 |
| 最近提现时间 | 最近一次提现申请时间 |
| 认证状态 | 未认证 / 审核中 / 已认证 / 认证失败 |
| 收款账户状态 | 未绑定 / 已绑定 / 审核中 / 异常 |

### 7.4 明细入口

钱包详情页应提供：

- 收益流水
- 结算记录
- 提现记录
- 收款账户
- 风险记录
- 人工调整记录
- 操作日志

## 8. 收益流水管理

### 8.1 页面目标

管理所有影响用户钱包金额的资金流水，并为客服、财务、审计提供追溯依据。

### 8.2 流水类型

```text
pending_income        待结算收益
settled_income        收益入账
withdraw_freeze       提现冻结
withdraw_success      提现成功
withdraw_failed       提现失败退回
manual_adjustment     人工调整
risk_deduction        风控扣减
refund_chargeback     退款扣回
bonus                 奖励发放
correction            系统修正
```

### 8.3 列表字段

| 字段 | 说明 |
| --- | --- |
| 流水号 | 平台资金流水编号 |
| 用户 ID | 所属 creator |
| 流水类型 | 入账 / 冻结 / 解冻 / 扣减 / 调整 |
| 金额 | 正数为增加，负数为减少 |
| 币种 | USD 等 |
| 状态 | 待处理 / 成功 / 失败 / 已撤销 |
| 来源类型 | 任务、结算单、提现单、人工调整等 |
| 来源单号 | 关联业务单号 |
| 操作人 | 系统或 Admin |
| 创建时间 | 流水创建时间 |
| 备注 | 规则或人工说明 |

### 8.4 关键规则

- 资金流水不可物理删除。
- 已成功流水不可直接修改，只能通过冲正流水修正。
- 每次余额变化必须有对应流水。
- 钱包余额应由流水可回溯计算。

### 9.1 页面目标

管理收益从“待结算”到“已到账”的过程。

### 9.2 结算单状态

```text
待生成
待审核
审核通过
已入账
已驳回
已作废
```

### 9.3 结算单字段

| 字段 | 说明 |
| --- | --- |
| 结算单号 | 平台结算编号 |
| 用户 ID | 结算对象 |
| 结算周期 | 如 2026-06-01 至 2026-06-30 |
| 收益来源 | 广告、任务、活动、商务合作等 |
| 预估金额 | 结算前金额 |
| 扣减金额 | 风控、退款、修正扣减 |
| 实际入账金额 | 最终入账金额 |
| 币种 | 默认 USD |
| 状态 | 当前结算状态 |
| 审核人 | 结算审核人 |
| 入账时间 | 收益进入钱包时间 |

### 9.4 功能要求

- 支持按周期生成结算单。
- 支持单笔或批量审核结算单。
- 支持驳回并填写原因。
- 支持入账前调整金额。
- 入账后生成收益入账流水。
- 入账后增加用户可提现余额。

## 10. 提现审核中心

### 10.1 页面目标

集中处理 creator 提交的提现申请，判断是否允许提现。

### 10.2 提现单状态

```text
待审核
审核通过
审核拒绝
待出款
出款处理中
出款成功
出款失败
已取消
```

### 10.3 列表字段

| 字段 | 说明 |
| --- | --- |
| 提现单号 | 平台提现编号 |
| 用户 ID | 申请人 |
| 提现金额 | 用户申请金额 |
| 手续费 | 平台或服务商手续费 |
| 实际到账金额 | 扣除手续费后的金额 |
| 币种 | 默认 USD |
| 收款方式 | PayPal / 银行账户 |
| 收款账户 | 脱敏展示 |
| 国家/地区 | 收款账户地区 |
| 风险等级 | 低 / 中 / 高 |
| 申请时间 | 用户提交时间 |
| 审核状态 | 待审核 / 通过 / 拒绝 |
| 付款状态 | 待出款 / 处理中 / 成功 / 失败 |
| 审核人 | Admin 操作人 |

### 10.4 审核详情

审核详情应展示：

- 用户基础信息
- 钱包金额结构
- 提现申请信息
- 收款账户信息
- 历史提现记录
- 最近收款账户变更记录
- 风控命中记录
- 关联收益来源
- 操作日志

### 10.5 审核动作

| 动作 | 结果 |
| --- | --- |
| 通过 | 提现单进入待出款 |
| 拒绝 | 释放冻结金额，提现单结束 |
| 打回修改 | 用户需更新收款账户或资料 |
| 标记风险 | 提现单进入风控复核 |
| 冻结钱包 | 阻止用户继续提现 |

### 10.6 审核规则

- 审核拒绝必须填写原因。
- 高风险提现不能批量通过。
- 大额提现建议二级审批。
- 收款账户新绑定或刚变更时，可触发提现冷却期。
- 审核通过不等于付款成功，仍需进入出款流程。

## 11. 出款管理

### 11.1 页面目标

管理提现审核通过后的实际付款流程。

### 11.2 出款状态

```text
待出款
出款处理中
服务商已受理
出款成功
出款失败
```

### 11.3 功能要求

## 12. 风控与冻结

### 12.1 页面目标

识别并处理异常钱包、异常收益和异常提现。

### 12.2 风险场景

- 异常点击或异常转化收益。
- 高频提现。
- 频繁更换收款账户。
- 多个账号绑定同一收款账户。
- 同一 IP 或设备创建大量账户。
- 退款率异常。
- 作弊任务收益。
- 高风险国家或地区收款。
- 收款账户信息与认证信息不一致。

### 12.3 风险等级

```text
低风险
中风险
高风险
禁止提现
```

### 12.4 冻结类型

| 类型 | 说明 |
| --- | --- |
| 钱包冻结 | 用户钱包整体不可提现 |
| 金额冻结 | 冻结指定金额 |
| 收益冻结 | 某笔收益不可结算 |
| 提现冻结 | 某笔提现暂停审核或出款 |

### 12.5 功能动作

- 冻结钱包
- 解冻钱包
- 冻结指定收益
- 释放冻结金额
- 拒绝提现
- 设置提现冷却期
- 标记高风险用户
- 添加风控备注

## 13. 人工调整

### 13.1 页面目标

支持在特殊情况下对用户钱包金额进行人工修正，但必须可控、可审计。

### 13.2 调整类型

```text
增加余额
扣减余额
冻结金额
释放冻结
补偿发放
违规扣减
冲正错误流水
```

### 13.3 调整申请字段

| 字段 | 说明 |
| --- | --- |
| 用户 ID | 调整对象 |
| 调整类型 | 增加 / 扣减 / 冻结 / 释放 |
| 调整金额 | 金额 |
| 币种 | 默认 USD |
| 调整原因 | 必填 |
| 关联单号 | 工单、结算单、提现单等 |
| 附件 | 可选 |
| 发起人 | 操作发起人 |
| 审批人 | 高权限角色 |
| 审批状态 | 待审批 / 通过 / 拒绝 |

### 13.4 调整规则

- 人工调整必须生成调整流水。
- 调整原因必填。
- 大额调整必须二级审批。
- 不能直接修改历史流水。
- 调整完成后需更新钱包金额快照。

## 14. 对账中心

### 14.1 页面目标

核对平台资金记录与第三方付款服务商、银行或 PayPal 账单，发现并处理差异。

### 14.2 对账对象

```text
平台提现记录
第三方付款服务商账单
银行实际出款记录
PayPal 出款记录
手续费记录
失败退款记录
```

### 14.3 对账维度

- 提现单号
- 服务商交易号
- 用户 ID
- 金额
- 币种
- 手续费
- 状态
- 出款时间
- 收款账户

### 14.4 对账结果

| 结果 | 说明 |
| --- | --- |
| 账平 | 平台与服务商记录一致 |
| 平台成功，服务商无记录 | 平台可能误更新状态 |
| 服务商成功，平台未成功 | 平台状态同步异常 |
| 金额不一致 | 金额或手续费存在差异 |
| 状态不一致 | 平台和服务商状态不同 |
| 重复出款 | 同一提现单出现多笔成功出款 |

### 14.5 差错处理

差错单应包含：

- 差错类型
- 关联提现单
- 平台金额
- 服务商金额

## 18. 异常场景

### 18.1 提现审核拒绝

处理逻辑：

```text
提现单状态改为审核拒绝
填写拒绝原因
释放冻结金额
金额退回可提现余额
通知 creator
记录操作日志
```

### 18.2 第三方出款失败

处理逻辑：

```text
出款状态改为失败
记录服务商失败代码和原因
释放冻结金额或进入待重试
根据失败原因决定是否允许重新出款
通知 creator
进入对账范围
```

### 18.3 重复出款风险

处理逻辑：

```text
同一提现单只能存在一笔成功出款
出款接口必须幂等
服务商请求必须使用唯一幂等键
重复成功时生成高优先级差错单
```

### 18.4 人工调账错误

处理逻辑：

```text
禁止修改原调账流水
通过反向冲正流水修复
记录冲正原因
关联原调整单
触发二级审批
```

### 18.5 对账金额不一致

处理逻辑：

```text
生成对账差错单
锁定相关提现单
人工核实平台流水与服务商账单
确认后执行状态同步、补款或调账
关闭差错单
```

## 22. 产品结论

Admin 钱包后台的本质是平台侧资金管理系统。它不只服务“查看数据”，更重要的是控制资金流转中的关键风险。

Creator 端链路是：

```text
看收益 -> 看余额 -> 发起提现 -> 查看到账
```

Admin 端链路是：

```text
查资金 -> 审结算 -> 审提现 -> 管出款 -> 控风险 -> 做对账 -> 留审计
```

因此 Admin 端最关键的产品能力是：

```text
提现审核
出款状态管理
风控冻结
人工调账
财务对账
权限审计
```

只要这几类能力定义清楚，后台就能支撑真实的资金运营、财务管理和异常追责。
